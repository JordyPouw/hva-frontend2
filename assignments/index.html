2.1: No jQuery, why?

  • Pros
    • The support of the community is great.
    • It makes DOM manipulation painless.
    • It makes basic animation a piece of cake.
    • Set selection is painless.
    • There are plug-ins.
  • Cons
    • The filesize is big.
    • You are being abstracted away from the vanilla JS.
    • Encourages chaining, which leads to code that is difficult to debug.
    • Performance is generally much slower
    • Quality of plug-ins is very variable.

2.2: Functionaliteiten jQuery

  • Asynchronous JavaScript and XML.
  • Events.
  • DOM manipulation.
  • Cross browser JS.
  • DOM selection.
